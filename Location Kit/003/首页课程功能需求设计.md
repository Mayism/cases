## 1. 需求概述

### 1.1 功能定位

首页课程功能是课程助手元服务的**核心入口**，为高校学生提供**即时性课程信息浏览**能力。通过今日/明日课程切换、课程列表展示、全部课程导航和快速添加课程的完整链路，帮助学生高效管理日常课程安排。

### 1.2 功能范围

首页课程相关功能包含以下4个核心模块：

| 模块 | 功能描述 | 优先级 |
|------|---------|--------|
| 今日课程 Tab | 展示当天所有课程安排，含已结束标记 | P0 |
| 明日课程 Tab | 展示明天所有课程安排 | P0 |
| 查看全部课程 | 跳转至课程中心查看周课表 | P0 |
| 添加课程 | 通过课程中心快速添加新课程 | P1 |

### 1.3 设计目标

| 目标 | 描述 |
|------|------|
| 即时可见 | 打开首页即可看到今日课程，零操作成本 |
| 快速切换 | 一键切换今日/明日课程，了解未来安排 |
| 信息完整 | 每门课程展示时间、地点、教师等关键信息 |
| 操作便捷 | 提供全部课程和添加课程的快速入口 |

## 2. 用户研究与场景分析

### 2.1 目标用户画像

| 属性 | 描述 |
|------|------|
| 年龄 | 18-25岁 |
| 身份 | 高校在校学生 |
| 技术水平 | 中等，习惯使用移动应用 |
| 使用频率 | 每日多次（课前查看、课间确认） |

## 4. 功能详细设计

### 4.1 今日课程 Tab

#### 4.1.1 功能描述

首页默认选中"今日课程"Tab，展示当天所有课程安排。系统根据当前日期和周数自动计算当天课程列表，并实时标记已结束的课程。

#### 4.1.2 数据展示规则

| 数据项 | 展示规则 |
|--------|---------|
| 课程列表 | 根据 `TimetableUtils.getCourseOverviewListByDate(timetable, today)` 计算得出 |
| 课程数量 | 标题显示"今日课程 N节"，N为课程列表长度 |
| 日期标签 | 格式 "M.D"（如 "3.27"），通过 `DateUtils.formatDateYmdDot()` 生成 |
| 时段标签 | 根据节次映射：上午（1-4节）/ 下午（5-8节）/ 晚上（9-10节） |
| 时间范围 | 根据节次对应的 `Period` 时间计算起止时间（如 "08:00-09:35"） |
| 已结束标记 | `finishedPeriodIndex >= course.periodEnd` 时，课程卡片变为灰度状态 |

#### 4.1.3 实时刷新机制

```
HomeTabVM 每秒执行一次 handleSecondTick():
  │
  ├── 更新 _minuteOfToday（当前分钟数）
  │    └── 影响 finishedPeriodIndex 计算 → 已结束标记实时更新
  │
  └── 检测日期变更
       ├── 同一天 → 仅更新分钟数
       └── 跨天 → 更新 _today + 刷新课程列表（todayCourseList + tomorrowCourseList）
```

#### 4.1.4 课程卡片信息结构

每张 `CourseOverviewCard` 展示以下信息：

```
┌──────────────────────────────────────────┐
│ ●  │  3.27           │  [课程名称]       │
│    │  上午            │  讲师：张三        │
│    │  08:00-09:35    │  教室：A301        │
│    │                 │              [装饰图]│
└──────────────────────────────────────────┘
  ↑         ↑                  ↑              ↑
 圆点    日期+时段+时间      课程详情        装饰元素
 (16px)   (80px宽)        (自适应宽度)
```

| 元素 | 规格说明 |
|------|---------|
| 左侧圆点 | 16×16vp，未结束时使用 `multi_color_09`，已结束使用 `icon_tertiary` |
| 日期标签 | "M.D" 格式，`Body_S` 字号，`font_secondary` 颜色 |
| 时段标签 | "上午"/"下午"/"晚上"，`Caption_M` 字号 |
| 时间范围 | "HH:MM-HH:MM" 格式，`Caption_M` 字号 |
| 课程名称 | `Caption_L` 字号加粗，最多2行，溢出省略 |
| 教师信息 | "讲师：XXX" 格式，`Caption_M` 字号，空值显示"未知" |
| 教室信息 | "教室：XXX" 格式，`Caption_M` 字号，空值显示"未知"，跑马灯滚动 |
| 卡片背景 | 未结束：`multi_color_09`（主色）；已结束：`background_secondary`（灰色） |
| 卡片高度 | 76vp |
| 卡片圆角 | 12vp |

### 4.2 明日课程 Tab

#### 4.2.1 功能描述

点击"明日课程"分段按钮，切换显示明天的课程安排。数据展示规则与"今日课程"一致，但不显示已结束标记。

#### 4.2.2 差异点

| 差异项 | 今日课程 | 明日课程 |
|--------|---------|---------|
| 已结束标记 | 根据当前时间实时标记已结束课程 | **不显示**已结束标记（全部显示为活跃状态） |
| 日期标签 | 当天日期 | 当天日期 + 1 |
| 默认选中 | 是 | 否 |
| 课程数据源 | `todayCourseList` | `tomorrowCourseList` |

#### 4.2.3 切换交互

```
用户点击 SegmentButtonV1 "明日课程"
  │
  ├── 触发 onIndexChanged(1)
  │
  ├── animateTo({ duration: 0 }) 屏蔽自带动画
  │
  ├── 更新 model.previewDay = RelativeDay.Tomorrow
  │
  └── UI 响应式刷新
       ├── courseList → _tomorrowCourseList
       ├── courseDateLabel → 明天日期
       ├── 标题 → "明日课程 N节"
       └── 课程卡片 → 全部显示为活跃状态
```

### 4.3 查看全部课程

#### 4.3.1 功能描述

提供从首页快速跳转至课程中心的入口，查看完整周课表视图。

#### 4.3.2 入口设计

| 入口位置 | 触发方式 | 导航目标 |
|---------|---------|---------|
| 课程概览卡片右上角"全部" | 点击 | `TimetablePage`（课程中心） |
| 空状态"全部课程"按钮 | 点击 | `TimetablePage`（课程中心） |
| 点击课程卡片 | 点击 | `CourseDetailPage`（课程详情） |

#### 4.3.3 课程中心页面结构

课程中心（TimetablePage）是完整的周课表视图：

```
┌─────────────────────────────────────┐
│  [←] 课程中心                       │
├─────────────────────────────────────┤
│  第N周          [日历] [+]          │
│  M月D日 - M月D日                    │
├─────────────────────────────────────┤
│  ┌─────────────────────────────┐    │
│  │ 课表网格（CourseSchedulePanel）│    │
│  │ 左侧：节次 + 时段            │    │
│  │ 顶部：星期 + 日期            │    │
│  │ 内容：课程色块卡片           │    │

[菜单"+" → 弹出]
```

| 元素 | 说明 |
|------|------|
| 周数标签 | "第N周"，通过 `Subtitle_L` 字号显示 |
| 日期范围 | "M月D日 - M月D日"，通过 `Subtitle_S` 字号显示 |
| 日历按钮 | 点击弹出周选择器半模态弹窗 |
| "+"按钮 | 点击弹出功能菜单（设置开学日期、添加课程） |

### 4.4 添加课程

#### 4.4.1 功能描述

#### 4.4.2 入口路径

```
首页 → 点击"全部" → 课程中心 → 点击"+" → 菜单选择"添加课程" → 课程表单页
```

#### 4.4.4 表单字段说明

| 字段 | 类型 | 必填 | 校验规则 | 输入方式 |
|------|------|------|---------|---------|
| 课程名称 | 文本 | 是 | ≤100字符，不能为空 | 文本输入框 |
| 课程周期 | 周掩码 | 是 | 至少选1周 | 半模态弹窗（周选择器） |
| 课程节数 | 星期+节次 | 是 | 选择星期和节次范围 | 半模态弹窗（节次选择器） |
| 教室地点 | 文本 | 否 | 无限制 | 文本输入框 |
| 任课教师 | 文本 | 否 | 无限制 | 文本输入框 |

#### 4.4.5 多时段支持

- 每门课程支持多个时段（CourseArrangement），至少保留1个
- 有多个时段时，每个时段右上角显示"×"删除按钮
- 点击"添加更多时间段 [+]"新增一个时段配置
- 各时段独立配置：课程周期、课程节数、教室地点、任课教师

#### 4.4.6 冲突校验

| 冲突类型 | 校验逻辑 | 提示信息 |
|---------|---------|---------|
| 自身时段重叠 | 同一课程内多个时段的时间和周期是否重叠 | "当前课程存在时段重叠" |
| 与其他课程冲突 | 与课表中已有课程的时间和周期是否冲突 | "当前课程与已有课程'XX'存在时段重叠" |

#### 4.4.7 保存流程

```
[点击"保存"]
  │
  ├── 校验课程名称非空
  │    └── 为空 → Toast "请输入课程名称"
  │
  ├── 校验自身时段无重叠
  │    └── 重叠 → Toast "当前课程存在时段重叠"
  │
  ├── 校验与其他课程无冲突
  │    └── 冲突 → Toast "当前课程与已有课程'XX'存在时段重叠"
  │
  ├── 自动分配颜色（ColorPalette循环取色）
  │
  ├── 保存到 PreferencesStore
  │
  ├── 更新 courseDataVersion（触发首页刷新）
  │
  └── 返回上一页
```

## 9. 状态设计

### 9.1 页面状态

| 状态 | 条件 | 展示内容 |
|------|------|---------|
| 今日有课程 | `todayCourseList.length > 0` | 课程列表（含已结束标记） |

| 明日有课程 | `tomorrowCourseList.length > 0` | 课程列表（全部活跃） |

### 9.2 课程卡片状态

| 状态 | 条件 | 视觉表现 |
|------|------|---------|
| 进行中/未开始 | `!isCourseFinished` | 主色背景 + 白色文字 + 白色装饰图 |
| 已结束 | `isCourseFinished && previewDay === Today` | 灰色背景 + 灰色文字 + 黑色装饰图 |

|------|---------|---------|
| 添加课程成功 | 保存课程后 `courseDataVersion` 自增 | HomeTabVM 监听到变化 → 重新加载课表 → 刷新课程列表 |
| 编辑课程成功 | 保存修改后 `courseDataVersion` 自增 | 同上 |
| 删除课程成功 | 删除后 `courseDataVersion` 自增 | 同上 |

### 10.3 ViewModel 数据管理（HomeTabVM）

```typescript
@ObservedV2
export class HomeTabVM {
  @Trace private _timetable: Timetable = TimetableUtils.deepClone(stores.pref.timetable);
  @Trace private _todayCourseList: CourseOverviewItem[] = [];
  @Trace private _tomorrowCourseList: CourseOverviewItem[] = [];
  @Trace public previewDay: RelativeDay = RelativeDay.Today;

  // 计算属性：根据当前Tab返回对应课程列表
  @Computed
  public get courseList(): CourseOverviewItem[] {
    return this.previewDay === RelativeDay.Today
      ? this._todayCourseList
      : this._tomorrowCourseList;
  }

  // 计算属性：已结束的节次索引
  @Computed
  public get finishedPeriodIndex(): number {
    const periods: Period[] = this._timetable.periods;
    const minuteOfDay: number = this._minuteOfToday;
    let index: number = -1;
    for (const period of periods) {
      if (minuteOfDay >= period.endMin) { index++; }
    }
    return index;
  }

  // 监听课程数据版本变化
  @Monitor('courseDataVersion')
  public onCourseDataVersionChange(): void {
    this._timetable = TimetableUtils.deepClone(stores.pref.timetable);
    this.refreshCourseList();
  }

  // 每秒刷新定时器
  private handleSecondTick(): void {
    const now: Date = new Date();
    this._minuteOfToday = DateUtils.getMinuteOfDay(now);
    if (!DateUtils.isSameDay(this._today, now)) {
      this._today = now;
      this.refreshCourseList();
    }
    const delay: number = 1000 - (Date.now() % 1000);
    setTimeout(() => this.handleSecondTick(), delay);
  }
}
```
