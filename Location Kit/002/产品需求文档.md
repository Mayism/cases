## 1. 产品概述

### 1.1 产品定位

课程助手是一款面向高校学生的 HarmonyOS 元服务，提供**课表管理**和**课堂签到**两大核心功能。学生可以快速查看今日/明日课程安排、管理学期课表、进行课堂签到，实现校园学习生活的数字化辅助。

### 1.2 核心功能模块

| 模块 | 功能描述 | 对应组件 |
|------|---------|---------|
| 首页 | 展示今日/明日课程概览、签到入口、轮播Banner | HomeTab |
| 课程中心 | 周课表展示、周切换、开学日期设置、添加/编辑/删除课程 | TimetablePage / CourseFormPage / CourseDetailPage |
| 课堂签到 | 签到码输入、位置验证、签到状态反馈 | CheckinPage |
| 我的 | 个人信息展示、华为账号关联/解绑 | MineTab |
| 服务卡片 | 桌面快捷入口（首页/打卡） | SchoolLifeWidgetCard |

### 1.3 功能结构树

```
课程助手
  ├── 首页
  │    ├── 轮播Banner
  │    ├── 签到入口（去签到按钮）
  │    └── 课程概览卡片
  │         ├── 今日课程 / 明日课程（分段切换）
  │         ├── 课程列表（含已结束标记）
  │         └── 全部课程入口
  ├── 课程中心
  │    ├── 周课表展示（CourseSchedulePanel）
  │    ├── 周选择器（WeekPicker）
  │    ├── 功能菜单
  │    │    ├── 设置开学日期
  │    │    └── 添加课程
  │    ├── 课程详情
  │    │    ├── 课程信息展示
  │    │    ├── 编辑课程
  │    │    └── 删除课程
  │    └── 课程表单（添加/编辑）

  │         │    ├── 教室地点
  │         │    └── 任课教师
  │         └── 保存
  ├── 课堂签到
  │    ├── 签到码输入（4位数字）
  │    ├── 签到确认
  │    ├── 签到成功/失败反馈
  │    └── 签到说明

  │    ├── 个人信息展示（头像、昵称、手机号）
  │    ├── 关联账号（华为账号手机号绑定）
  │    └── 取消关联
  └── 服务卡片（1x2）
       ├── 首页入口
       └── 打卡入口
```

## 2. 业务使用场景

### 2.1 高优先级场景

| 编号 | 场景名称 | 场景描述 | 用户目标 |
|------|---------|---------|---------|
| S-H01 | 查看今日课程 | 学生早晨打开元服务，快速查看今天有哪些课、第几节、在哪个教室 | 掌握当日课程安排，避免迟到或走错教室 |
| S-H02 | 课堂签到 | 课堂上老师公布4位签到码，学生输入签到码完成签到 | 完成课堂考勤，证明出勤 |
| S-H03 | 添加新课程 | 新学期开始或课程变动时，学生手动录入课程信息 | 建立个人课表数据，支撑课表展示 |
| S-H04 | 查看周课表 | 学生想了解本周或某一周的完整课程安排 | 规划一周学习时间 |

### 2.2 中优先级场景

| 编号 | 场景名称 | 场景描述 | 用户目标 |
|------|---------|---------|---------|
| S-M01 | 查看明日课程 | 学生晚上想确认明天有哪些课 | 提前准备明天所需教材和材料 |
| S-M02 | 编辑课程信息 | 课程教室变更或教师更换时，修改已有课程信息 | 保持课表数据准确 |
| S-M03 | 删除课程 | 课程退选或取消时，从课表中移除 | 保持课表数据准确 |
| S-M04 | 设置开学日期 | 新学期开始时，设置学期起始日期 | 让系统正确计算当前周数 |
| S-M05 | 关联华为账号 | 用户首次使用签到功能前，需关联华为账号手机号 | 解锁签到功能 |

### 2.3 低优先级场景

| 编号 | 场景名称 | 场景描述 | 用户目标 |
|------|---------|---------|---------|
| S-L01 | 查看课程详情 | 学生想查看某门课程的完整信息（时间、地点、教师） | 了解课程详细信息 |
| S-L02 | 取消账号关联 | 用户不再使用签到功能或更换手机号时 | 解除手机号绑定 |
| S-L03 | 通过服务卡片快速签到 | 用户在桌面卡片上直接点击"打卡"跳转到签到页 | 快速进入签到流程 |
| S-L04 | 通过服务卡片返回首页 | 用户在桌面卡片上点击"首页"跳转到应用首页 | 快速查看课程信息 |

## 3. 业务流程

### 3.1 应用启动流程

```
[启动应用]
    │
    ├── 检查隐私协议状态
    │    ├── 未同意 → 等待用户同意（阻塞UI交互）
    │    └── 已同意 → 继续
    │
    ├── 初始化课表数据
    │    ├── 本地无课表数据 → 创建默认课表（25周、10节课、当天为锚点）
    │    └── 本地有课表数据 → 加载已有数据
    │
    ├── 初始化认证状态
    │    ├── 有sessionToken → 静默登录 → 刷新用户信息
    │    ├── 无sessionToken → 尝试华为账号静默登录
    │    │    ├── 成功 → 刷新用户信息 → 检查手机号绑定
    │    │    │    ├── 已绑定手机号 → 认证状态=已登录
    │    │    │    └── 未绑定手机号 → 登出 → 认证状态=未登录
    │    │    └── 失败 → 清除用户信息 → 认证状态=未登录
    │    └── 认证状态=未知 → 同上流程
    │
    ├── 处理意图任务（来自服务卡片）
    │    ├── GO_TO_HOME_PAGE → 切换到首页Tab
    │    ├── GO_TO_CHECKIN_PAGE → 跳转签到页
    │    └── NONE → 无操作
    │
    └── [显示主界面（首页Tab）]
```

### 3.2 课程管理流程

```
[课程管理主流程]
    │
    ├── 查看课表
    │    ├── 首页 → 今日/明日课程概览
    │    │    ├── 点击课程卡片 → 课程详情页
    │    │    └── 点击"全部" → 课程中心页
    │    │
    │    └── 课程中心 → 周课表视图
    │         ├── 切换周数 → 周选择器
    │         ├── 点击课程卡片 → 课程详情页
    │         └── 功能菜单
    │              ├── 设置开学日期 → 日历选择器
    │              └── 添加课程 → 课程表单页
    │
    ├── 添加课程
    │    ├── 输入课程名称（必填，≤100字）
    │    ├── 添加课程时段（至少1个）
    │    │    ├── 选择课程周期（周掩码选择）
    │    │    ├── 选择课程节数（星期+节次范围）
    │    │    ├── 输入教室地点（选填）
    │    │    └── 输入任课教师（选填）
    │    ├── 可添加多个时段
    │    ├── 冲突校验
    │    │    ├── 自身时段冲突 → 提示"当前课程存在时段重叠"
    │    │    └── 与其他课程冲突 → 提示"与已有课程'XX'存在时段重叠"
    │    └── 保存 → 返回课程中心
    │
    ├── 编辑课程
    │    ├── 从课程详情页进入
    │    ├── 加载已有课程数据
    │    ├── 修改后保存（同添加流程的校验逻辑）
    │    └── 保存 → 返回课程详情
    │
    └── 删除课程
         ├── 从课程详情页触发
         ├── 确认弹窗："课程删除后将会从课程中心移除且无法恢复，是否删除？"
         ├── 确认 → 删除并返回上级
         └── 取消 → 留在当前页
```

### 3.3 签到流程

```
[签到主流程]
    │
    ├── 点击"去签到"按钮
    │    ├── 未关联账号 → Toast提示"请关联账号后使用签到"
    │    ├── 网络异常 → Toast提示"网络繁忙，请稍后重试"
    │    └── 已关联账号 → 进入签到页
    │
    ├── 签到页初始化
    │    ├── 请求签到信息（获取课程ID、课程名称、签到状态）
    │    ├── 已签到 → 显示签到成功状态（课程名+签到成功图标）
    │    └── 未签到 → 显示签到码输入界面
    │
    ├── 输入4位签到码
    │    ├── 输入过程中 → 实时更新签到码
    │    ├── 输入完成4位 → 激活"确定"按钮
    │    └── 输入不足4位 → "确定"按钮禁用
    │
    ├── 点击"确定"提交签到
    │    ├── 请求位置权限
    │    │    ├── 权限拒绝 → Toast提示"位置权限获取失败"
    │    │    └── 权限授予 → 获取当前位置
    │    │         ├── 位置获取失败 → Toast提示"位置信息获取失败"
    │    │         └── 位置获取成功 → 提交签到请求
    │    │
    │    ├── 签到成功 → 显示签到成功状态
    │    └── 签到失败（签到码错误）→ 显示错误提示"请输入正确的签到码"
    │
    └── 签到说明
         ├── 1、正确输入任课老师公布的四位数签到码
         ├── 2、未签到成功的同学，请找老师补签
         └── 3、开发模式下签到码默认为: 1111
```

### 3.4 账号关联流程

```
[账号关联主流程]
    │
    ├── 未关联状态
    │    ├── 显示默认头像 + "华为用户" + "关联账号"按钮
    │    └── 点击"关联账号"
    │         ├── 拉起华为账号授权
    │         │    ├── 用户取消 → 无操作
    │         │    ├── 授权失败 → 弹窗提示"授权服务拉起失败"
    │         │    │    ├── 取消 → 无操作
    │         │    │    └── 模拟关联 → 使用随机UUID模拟绑定
    │         │    └── 授权成功 → 获取手机号 → 绑定 → 刷新用户信息
    │         └── 关联成功 → 显示用户信息
    │
    └── 已关联状态
         ├── 显示用户头像 + 昵称 + 脱敏手机号（138****1234）
         └── 点击"取消关联"
              ├── 确认弹窗："请确认是否解除账号关联？"
              ├── 确认 → 登出 → 回到未关联状态
              └── 取消 → 无操作
```

## 4. 操作步骤

### 4.1 查看今日/明日课程

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 打开元服务 | 进入首页，默认展示"今日课程" |
| 2 | 查看课程列表 | 显示今日课程卡片列表，每张卡片包含课程名、时段标签、时间范围、教室、教师、颜色标识 |
| 3 | 已结束课程标记 | 当前时间已过的课程节次，视觉上标记为已结束 |
| 4 | 切换到"明日课程" | 点击分段按钮"明日课程"，显示明日课程列表 |
| 5 | 无课程时 | 显示空状态图标 + "暂无预课程安排" + "全部课程"按钮 |
| 6 | 点击课程卡片 | 跳转到课程详情页 |
| 7 | 点击"全部" | 跳转到课程中心页 |

### 4.2 添加课程

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 在课程中心页点击"+"按钮 | 弹出功能菜单 |
| 2 | 点击"添加课程" | 进入课程表单页，标题显示"添加课程" |
| 3 | 输入课程名称 | 在"课程名称"行输入，最多100字符 |
| 4 | 设置课程周期 | 点击"课程周期"行，弹出周选择器，支持全选/单周/双周/自定义 |
| 5 | 设置课程节数 | 点击"课程节数"行，弹出星期+节次范围选择器 |
| 6 | 输入教室地点 | 在"教室地点"行输入（选填） |
| 7 | 输入任课教师 | 在"任课教师"行输入（选填） |
| 8 | 添加更多时段 | 点击"添加更多时间段"，新增一个时段配置 |
| 9 | 删除时段 | 当有多个时段时，点击时段右上角"×"删除（至少保留1个） |
| 10 | 点击"保存" | 校验通过后保存课程，返回课程中心；校验失败则Toast提示错误信息 |

### 4.3 编辑/删除课程

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 在课程详情页点击"编辑" | 进入课程表单页，标题显示"编辑课程"，预填充课程数据 |
| 2 | 修改课程信息 | 同添加课程流程 |
| 3 | 点击"保存" | 保存修改，返回课程详情页 |
| 4 | 在课程详情页点击"删除" | 弹出确认弹窗 |
| 5 | 点击"删除"确认 | 删除课程，返回上级页面，Toast提示"删除成功" |
| 6 | 点击"取消" | 关闭弹窗，留在当前页 |

### 4.4 课堂签到

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 在首页点击"去签到" | 若已关联账号，进入签到页；若未关联，Toast提示 |
| 2 | 查看签到说明 | 页面底部显示3条签到说明 |
| 3 | 输入4位签到码 | 在4个输入框中依次输入数字 |
| 4 | 点击"确定" | 系统请求位置权限并获取位置，提交签到 |
| 5 | 签到成功 | 显示课程名称 + 签到成功图标 + "签到成功"文字 |
| 6 | 签到码错误 | 输入框下方显示红色错误提示"请输入正确的签到码"，清空输入 |

### 4.5 设置开学日期

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 在课程中心页点击"+"按钮 | 弹出功能菜单 |
| 2 | 点击"设置开学日期" | 弹出日历选择器 |
| 3 | 选择日期 | 确认后更新学期锚点日期，重新计算当前周数 |
| 4 | 自动更新 | 课表数据自动刷新，周数标签更新 |

### 4.6 关联/取消关联账号

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 切换到"我的"Tab | 显示个人信息区域 |
| 2 | 未关联时点击"关联账号" | 拉起华为账号授权流程 |
| 3 | 授权成功 | 显示用户头像、昵称、脱敏手机号 |
| 4 | 已关联时点击"取消关联" | 弹出确认弹窗 |
| 5 | 确认取消关联 | 登出，回到未关联状态 |

## 5. 约束规则

### 5.1 环境约束

| 约束项 | 要求 |
|--------|------|
| DevEco Studio版本 | 6.0.2 Release 及以上 |
| HarmonyOS SDK版本 | 6.0.2 Release SDK 及以上 |
| 设备类型 | 华为手机（包括双折叠和阔折叠） |
| 系统版本 | HarmonyOS 5.1.0(18) 及以上 |

### 5.2 权限约束

| 权限 | 用途 | 申请时机 |
|------|------|---------|
| ohos.permission.INTERNET | 网络访问（签到API、账号服务） | 安装时自动授予 |
| ohos.permission.VIBRATE | 振动反馈 | 安装时自动授予 |
| 位置权限 | 签到时获取用户位置信息 | 签到时动态申请 |

### 5.3 数据约束

| 约束项 | 规则 |
|--------|------|
| 课程名称最大长度 | 100 字符（`MAX_COURSE_NAME_LENGTH`） |
| 学期最大周数 | 31 周（`weekMask` 为32位整数，最高位为符号位，有效31位） |
| 默认学期周数 | 25 周（`DEFAULT_TOTAL_WEEKS`） |
| 默认课程节数 | 10 节（上午4节 + 下午4节 + 晚上2节） |
| 签到码长度 | 4 位数字 |
| 课程时段 | 每门课程至少 1 个时段（arrangement），可添加多个 |
| 课程颜色 | 系统自动分配，从调色板循环取色 |
| 学期标签格式 | "YYYY-YYYY学年 第N学期"（根据锚点日期自动生成） |

### 5.4 业务规则

| 规则编号 | 规则描述 |
|----------|---------|
| BR-01 | 课程名称为必填项，为空时保存提示"请输入课程名称" |
| BR-02 | 同一课程内多个时段不可时间重叠，重叠时提示"当前课程存在时段重叠" |
| BR-03 | 不同课程的时段不可时间冲突，冲突时提示"当前课程与已有课程'XX'存在时段重叠" |
| BR-04 | 课程时段至少保留1个，不可全部删除 |
| BR-05 | 课程删除后不可恢复，需二次确认 |
| BR-06 | 签到功能需先关联华为账号，未关联时点击签到提示"请关联账号后使用签到" |
| BR-07 | 签到需获取位置权限，权限拒绝时提示"位置权限获取失败" |

## 7. 数据模型

### 7.1 核心数据结构

```
Timetable（课表）
├── id: string                    // 课表唯一标识
├── semester: Semester            // 学期信息
│   ├── label: string            // 学期描述（如"2025-2026学年 第2学期"）
│   ├── anchorTimeMs: number      // 锚点时间戳（开学后第一节课所在日期）
│   └── weekCount: number         // 学期总周数（1-31）
├── periods: Period[]             // 节次时间表
│   └── Period
│       ├── dayPart: DayPart      // 时段（MORNING/AFTERNOON/EVENING）
│       ├── startMin: number      // 起始分钟数（0-1439）
│       └── endMin: number         // 结束分钟数（0-1439）
└── courses: Course[]             // 课程列表
    └── Course
        ├── id: string            // 课程唯一标识
        ├── name: string          // 课程名称（≤100字符）
        ├── color: string         // 课程颜色（系统自动分配）
        └── arrangements: CourseArrangement[]  // 课程安排列表
            └── CourseArrangement
                ├── id: string            // 安排唯一标识
                ├── courseId: string      // 所属课程ID
                ├── dayOfWeek: number     // 星期几（0=周日, 1=周一...6=周六）
                ├── periodStart: number   // 起始节次索引（0=第1节）
                ├── periodEnd: number     // 结束节次索引
                ├── location: string      // 教室地点
                ├── teacher: string        // 任课教师
                └── weekMask: number      // 周选择掩码（位运算）

AuthStatus（认证状态）
├── Unknown = 0    // 未知
├── SignedOut = 1  // 未登录
└── SignedIn = 2   // 已登录

RelativeDay（相对日期）
├── Today = "TODAY"      // 今天
└── Tomorrow = "TOMORROW" // 明天

FormMode（表单模式）
├── Create = "CREATE"   // 创建
└── Edit = "EDIT"        // 编辑

IntentTask（意图任务）
├── NONE = "NONE"                        // 无任务
├── GO_TO_HOME_PAGE = "GO_TO_HOME_PAGE"  // 跳转首页
└── GO_TO_CHECKIN_PAGE = "GO_TO_CHECKIN_PAGE" // 跳转签到页
```

### 7.2 数据存储

| 存储类型 | 存储内容 | 实现方式 |
|---------|---------|---------|
| PreferencesStore | 课表数据、会话令牌 | 持久化存储，flushSync同步写入 |
| UIStore | 用户头像、昵称、手机号绑定状态 | 可驱动UI的轻量级持久化存储 |
| CacheStore | 认证状态、活跃Tab、课程数据版本号、意图任务 | 运行时缓存 |
