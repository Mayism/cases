# 医院查看地址和导航功能 技术实现

> 版本：1.0.0 | 平台：HarmonyOS | 更新日期：2026-04-29

## 1. 技术架构

本功能是 `features/appointment` 模块内的子功能，嵌入在现有预约挂号流程中。

### 模块结构
```
features/appointment/src/main/ets/
├── model/
│   └── HospitalModel.ets          ← 新增 address、latitude、longitude 字段
├── viewmodel/
│   └── AppointmentViewModel.ets   ← 新增 selectHospitalPosition 方法
├── components/
│   ├── HospitalSelectPage.ets     ← 新增地址展示行和导航按钮
│   └── AppointmentOrderPage.ets   ← 修改就诊地址展示格式
└── resources/rawfile/
    ├── get.appointment.hospital.json  ← 新增 address、latitude、longitude 字段
    └── post.appointment.order.json    ← 新增 hospitalInfo.address 字段
```

### 组件层级
```
AppointmentEntryPage（入口页）
  └─ HospitalSelectPage（医院选择页）
       ├─ hospitalCardBuilder
       │    ├─ 地址展示行（Image + Text）  ← 新增

AppointmentOrderPage（订单详情页）
  └─ 就诊地址 Text（hospitalName + address）← 修改展示格式
```

## 2. 页面组件实现

### 2.1 HospitalSelectPage - 医院卡片组件

**组件名称**：`HospitalSelectPage`
**文件路径**：`features/appointment/src/main/ets/components/HospitalSelectPage.ets`

**HospitalCardParam 接口**（需新增字段）：
```typescript
interface HospitalCardParam {
  index: number,       // 医院在列表中的索引，用于导航定位
  picUrl: string,
  name: string,
  address: string      // ← 新增：医院地址文字
}
```

**hospitalCardBuilder 中新增的地址展示行**：
```
布局结构（在名称 Text 下方）：
Row({ space: 4 })
├─ Image($r('app.media.ic_local_fill'))   // 定位图标，12x12，次要颜色
└─ Text(param.address)                     // 地址文字，12px，三级文字色，layoutWeight(1)
```

**hospitalCardBuilder 中新增的导航按钮**（卡片右侧）：
```
Image($r('app.media.ic_location_north_up_right_circle_fill'))
  .width(24)
  .height(24)
  .fillColor($r('sys.color.ohos_id_color_secondary'))
  .onClick(() => {
    this.appointmentViewModel.selectHospitalPosition(param.index)
  })
```

**状态管理**：
- 使用 `@ComponentV2` + `@Local` 管理组件内部状态
- `appointmentViewModel` 通过 `@Local` 注入单例
- 医院列表数据通过 `ForEach` 渲染，key 为 `item.id.toString()`

**事件处理**：
- 导航按钮点击 → `appointmentViewModel.selectHospitalPosition(index)` → 跳转地图页面
- 卡片整体点击 → `appointmentViewModel.selectHospital(item)` → 进入科室选择

### 2.2 AppointmentOrderPage - 就诊地址展示

**组件名称**：`AppointmentOrderPage`
**文件路径**：`features/appointment/src/main/ets/components/AppointmentOrderPage.ets`

**就诊地址行修改**：
```typescript
// 修改前
Text(`就诊地址：${...hospitalInfo.name}`)

// 修改后（拼接地址详情）
Text(`就诊地址：${this.appointmentViewModel.appointmentInfoModel.hospitalInfo.name} ${this.appointmentViewModel.appointmentInfoModel.hospitalInfo.address}`)
  .fontSize(12)
  .fontColor($r('sys.color.ohos_id_color_text_tertiary'))
  .margin({ top: 8 })
```

**数据来源**：
- `hospitalInfo.name`：来自 `HospitalModel`（用户在医院选择页选择）
- `hospitalInfo.address`：来自订单创建接口响应 `data.hospitalInfo.address`（在 `createAppointmentOrder` 中赋值）

### 2.3 地图导航页面（需新建）

**组件名称**：需创建，如 `HospitalMapPage`

- 接收参数：医院的 `latitude`、`longitude`、`name`、`address`

## 3. 关键API使用

### 3.1 HarmonyOS Kit
- **[核心] `@kit.MapKit`**：使用 `MapComponent` 展示地图，`mapController` 控制地图交互，需导入地图相关组件实现地图展示和导航功能
- **[核心] `@kit.LocationKit`**：使用 `geoLocationManager` 获取用户当前位置，用于路线规划起点
- **[辅助] `@kit.AbilityKit`**：已在项目中导入，用于获取 UIAbilityContext

### 3.2 第三方依赖
- 无新增第三方依赖

## 4. 数据模型

### 4.1 HospitalModel（需新增字段）[核心]
```typescript
// 文件：features/appointment/src/main/ets/model/HospitalModel.ets
@ObservedV2
export class HospitalModel {
  @Trace id: string = '';
  @Trace name: string = ''
  @Trace address: string = ''      // [核心] 医院地址文字
  @Trace picUrl: string = ''
  @Trace latitude: number = 0      // [核心] 纬度坐标
  @Trace longitude: number = 0     // [核心] 经度坐标
}
```

### 4.2 AppointmentInfoModel 关联 [辅助]
```typescript
// hospitalInfo 字段类型为 HospitalModel
// address 属性同时从两个来源赋值：
// 1. 用户选择医院时（selectHospital）→ 携带 HospitalModel 的 address
// 2. 创建订单后（createAppointmentOrder）→ 从接口响应 data.hospitalInfo.address 覆盖
```

## 5. 网络请求

### 5.1 获取医院列表 [核心]
- **请求**：`GET /appointment/hospital`
- **响应字段变更**：需新增 `address`（string）、`latitude`（number）、`longitude`（number）
- **映射**：响应数据直接映射为 `HospitalModel[]`

### 5.2 创建挂号订单 [核心]
- **请求**：`POST /appointment/order`
- **响应字段变更**：`hospitalInfo` 对象需新增 `address` 字段
- **使用方式**：
  ```typescript
  this.appointmentInfoModel.hospitalInfo.address = data.hospitalInfo.address
  ```
