# 医院查看地址和导航功能 需求PRD

> 版本：1.0.0 | 平台：HarmonyOS | 更新日期：2026-04-29

## 1. 业务场景

用户在医院预约挂号流程中，需要在医院选择页面查看每个医院的具体地址信息，并通过跳转花瓣地图导航功能规划前往医院的路线。挂号成功后，在订单详情页需要展示完整的就诊地址信息（包含医院名称和地址详情），方便用户确认就诊位置。

## 2. 业务目标

- 提供医院地址信息的可视化展示，帮助用户了解医院具体位置
- 提供地图导航入口，方便用户规划前往医院的出行路线
- 在订单详情页展示完整就诊地址，减少用户因地址不明确导致的就诊延误

## 3. 功能需求

### 3.1 医院选择页面 - 地址展示 \[P0]

- 在医院选择列表的每个医院卡片中，展示医院的详细地址文字
- 地址文字前显示定位图标（`ic_local_fill`），使用次要文字颜色
- 地址文字使用 12px 字号，单行展示，超长部分省略

### 3.2 医院选择页面 - 地图导航入口 \[P0]

- 在每个医院卡片右侧提供导航按钮，使用地图方向图标（`ic_location_north_up_right_circle_fill`）
- 导航按钮尺寸为 24x24，使用次要图标颜色
- 点击导航按钮后，跳转到地图页面展示该医院的地理位置
- 地图页面需支持查看医院在地图上的位置标记（基于经纬度坐标）
- 地图页面需提供跳转花瓣地图导航功能，在花瓣地图上展示从用户当前位置到医院的路线

### 3.3 挂号订单详情页 - 就诊地址展示 \[P0]

- 在挂号订单详情中展示完整的就诊地址信息
- 展示格式为"就诊地址：{医院名称} {地址详情}"
- 地址信息来源于挂号订单创建接口返回的 `hospitalInfo.address` 字段
- 地址详情包含具体的就诊位置信息（如"门诊楼3楼615诊室"）

### 3.4 医院数据模型 - 地址与坐标字段 \[P0]

- 医院数据模型需包含 `address`（地址文字）、`latitude`（纬度）、`longitude`（经度）字段
- 医院列表接口返回数据需包含上述字段
- 经纬度坐标用于地图定位和导航功能

## 5. 核心技术方案

- 使用 `@kit.MapKit` 提供地图展示和导航能力
- 使用 `@kit.LocationKit` 获取用户当前位置用于路线规划
- 使用 `@ObservedV2/@Trace` 装饰器管理医院数据模型的响应式状态
- 地图页面通过路由表注册，从医院选择页面导航跳转

## 6. 数据流向

```
医院列表接口 → HospitalModel(address, latitude, longitude)
    ↓
HospitalSelectPage（展示地址 + 提供导航入口）
    ↓ 点击导航按钮
地图页面（接收经纬度坐标 → 展示地图标记 → 提供导航）

订单创建接口 → hospitalInfo.address
    ↓
AppointmentOrderPage（展示就诊地址）
```

## 7. 交互流程

### 7.1 查看医院地址 \[P0]

1. 用户进入医院选择页面
2. 页面加载医院列表数据（含地址、坐标）
3. 每个医院卡片展示医院名称和地址信息
4. 用户可查看各医院地址进行比较选择

### 7.2 地图导航 \[P0]

1. 用户在医院卡片上点击导航按钮（右侧地图图标）
2. 系统跳转至地图页面
3. 地图页面根据医院经纬度定位并展示位置标记
4. 用户可在地图页面发起导航，跳转花瓣地图规划从当前位置到医院的路线

### 7.3 查看就诊地址 \[P0]

1. 用户完成挂号支付流程
2. 系统跳转至挂号订单详情页
3. 订单详情中展示完整就诊地址（医院名称 + 地址详情）
4. 用户确认就诊位置信息

## 8. 功能边界与限制条件

- \[P0] 医院列表数据必须包含 address、latitude、longitude 字段，否则地址展示和导航功能不可用
- \[P0] 地图导航功能需要位置权限（ohos.permission.LOCATION），需在 module.json5 中声明
- \[P0] 导航按钮点击时，需校验经纬度坐标有效性（非0值），无效时提示用户
- \[P1] 地址文字超长时应省略显示，避免布局溢出
- \[P1] 订单详情中地址详情可能为空（如新创建订单未获取到），此时仅显示医院名称

## 9. 验收标准与成功指标

- \[P0] 功能完整性：医院选择页面正确展示各医院地址信息和导航按钮
- \[P0] 功能完整性：点击导航按钮可正常跳转至地图页面并展示医院位置
- \[P0] 功能完整性：挂号订单详情页正确展示就诊地址信息
- \[P0] 数据完整性：医院数据模型包含 address、latitude、longitude 字段且数据正确映射
- \[P1] 交互体验：地址文字超长时不影响卡片布局
