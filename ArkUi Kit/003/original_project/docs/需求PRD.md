# 钱包功能需求PRD

> 版本：1.0.0 | 平台：HarmonyOS | 更新日期：2026-04-30

## 1. 业务场景
商场模板应用中，为注册会员提供电子钱包服务。会员可在"我的"中心查看钱包余额，进入钱包详情页查看充值和消费变动明细，通过刮刮卡密钥进行充值。在停车缴费场景中，会员可开启钱包余额抵扣停车费用，实现积分、优惠券、钱包的组合支付。

## 2. 业务目标
- 提供会员虚拟钱包服务，增强用户粘性和消费体验
- 支持充值和消费记录追踪，提升财务透明度
- 在停车缴费场景中提供多种支付方式组合（积分+优惠券+钱包），降低用户支付成本

## 3. 功能需求

### 3.1 钱包入口展示 [P0]
- 在"我的"页面 mineCard 区域展示"我的钱包"卡片
- 卡片显示钱包图标、"我的钱包"文字和当前余额（格式：￥xxx.xx）
- 点击卡片跳转至钱包详情页

### 3.2 钱包详情页 [P0]
- 顶部展示钱包余额，格式为 ￥xxx.xx（保留两位小数）
- 提供"充值"按钮，点击跳转充值页面
- 支持按月份筛选变动记录（年+月双列滚轮选择器）
- 支持按类型筛选：全部、充值、支出（Tab标签切换）
- 变动明细列表展示每条记录的名称、时间和金额
- 充值记录金额显示为绿色（主题色）带"+"前缀，支出记录显示默认色带"-"前缀
- 无数据时显示"暂无历史记录"空状态

### 3.3 充值功能 [P0]
- 充值页面包含序列号输入框和密钥输入框
- 序列号和密钥均不能为空，需进行输入校验
- 输入校验不通过时展示对应 Toast 提示
- 充值成功后余额增加 100 元，自动插入一条充值记录
- 返回钱包详情页时展示"充值成功"Toast

### 3.4 缴费详情钱包支付 [P0]
- 在缴费详情页展示"电子钱包支付"卡片
- 卡片显示电子钱包名称和当前余额
- 提供 Toggle 开关控制是否使用钱包支付
- 余额为 0 时 Toggle 开关不可用（置灰）
- 开启钱包支付后自动计算可抵扣金额（余额 >= 剩余应付则全额抵扣，否则使用全部余额）
- 显示本次使用的钱包抵扣金额
- 支付成功后扣减对应余额，并生成一条支出变动记录

### 3.5 会员一码通描述更新 [P1]
- 会员一码通页面的功能描述中包含"电子钱包支付"文字

## 5. 交互流程

### 5.1 查看钱包余额 [P0]
1. 用户进入"我的"页面
2. 在 mineCard 区域看到"我的钱包"卡片和余额
3. 点击卡片跳转至 MyWalletPage

### 5.2 查看变动明细 [P0]
1. 进入钱包详情页，默认显示当月全部记录
2. 点击年月选择器，弹出半模态 Sheet 展示双列滚轮
3. 选择年月后点击"确认"，列表按所选月份筛选
4. 点击"取消"恢复之前的选择
5. 点击"全部/充值/支出"Tab 标签按类型筛选

### 5.3 充值流程 [P0]
1. 在钱包详情页点击"充值"按钮
2. 跳转至 RechargeWalletPage
3. 输入序列号（数字键盘）和密钥
4. 点击"立即充值"
5. 校验输入，通过后返回钱包详情页
6. 余额增加 100 元，明细列表顶部新增一条充值记录
7. Toast 提示"充值成功~"

### 5.4 停车缴费使用钱包 [P0]
1. 用户在缴费详情页看到"电子钱包支付"卡片
2. 余额 > 0 时可开启 Toggle 开关
3. 开启后系统自动计算可抵扣金额并显示
4. "还需支付"金额实时更新（扣除钱包抵扣部分）
5. 点击"立即支付"完成支付
6. 支付成功后余额扣减，新增一条支出记录

### 5.5 充值取消 [P1]
1. 在充值页面点击"立即充值"
2. 序列号为空时 Toast 提示"序列号不能为空"
3. 密钥为空时 Toast 提示"密钥不能为空"

## 6. 功能边界与限制条件

- [P0] 钱包余额初始化为 200 元
- [P0] 每次充值固定增加 100 元（模拟充值）
- [P0] 钱包余额不可为负数
- [P0] 缴费时钱包抵扣金额不超过剩余应付金额
- [P0] 钱包余额存储在本地，不上传服务器
- [P1] 月份筛选支持 1990 年至 2030 年范围
- [P1] 金额显示使用中文格式，保留两位小数
- [P1] 变动明细按时间倒序排列（最新在前）
