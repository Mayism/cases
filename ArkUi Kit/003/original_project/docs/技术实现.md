# 钱包功能技术实现

> 版本：1.0.0 | 平台：HarmonyOS | 更新日期：2026-04-30

## 1. 技术架构

### 1.1 模块结构
钱包功能作为 `business_mine` 模块的子功能实现，涉及跨模块交互：

```
├── features/business_mine/          （钱包页面和ViewModel）
│   ├── pages/MyWalletPage.ets       （钱包详情页）
│   ├── pages/RechargeWalletPage.ets （充值页面）
│   ├── viewmodel/MyWalletViewModel.ets    （钱包核心逻辑）
│   ├── viewmodel/RechargeWalletViewModel.ets （充值输入状态）
│   ├── constants/WalletConstants.ets      （Tab筛选常量）
│   └── pages/MinePage.ets           （钱包入口卡片）
├── features/business_home/          （缴费详情钱包支付卡片）
│   ├── pages/PayDetailPage.ets      （钱包卡片UI）
│   └── viewmodel/PayDetailViewModel.ets （钱包支付逻辑）
├── commons/lib_common/              （共享类型、API、全局状态）
│   ├── models/ResponseModel.ets     （IWallet/Wallet 类型）
│   ├── constants/RouterMap.ets      （路由枚举）
│   ├── constants/Common.ets         （API URL 常量）
│   ├── utils/GlobalUtils.ets        （全局余额状态）
│   ├── httprequest/HttpRequest.ets  （钱包明细请求）
│   ├── httprequest/HttpRequestApi.ets （钱包API封装）
│   ├── httprequest/AxiosHttp.ets    （Mock路由）
│   └── mock/MockData.ets            （模拟数据）
└── products/phone/                  （全局钱包消费事件监听）
    ├── viewmodel/IndexViewModel.ets （事件监听/分发）
    └── pages/Index.ets              （生命周期清理）
```

### 1.2 组件层级
```
MinePage（我的页面）
  └── mineCard Builder
       └── 钱包卡片 Row（点击跳转 MyWalletPage）

MyWalletPage（钱包详情）
  ├── NavHeaderBar（标题栏）
  ├── 余额展示 Row + 充值按钮
  ├── 月份筛选 Row + TextPicker Sheet
  ├── Tab 筛选 Row（全部/充值/支出）
  └── 变动明细 List / 空状态 EmptyComp

RechargeWalletPage（充值页）
  ├── NavHeaderBar
  ├── 序列号 TextInput
  ├── 密钥 TextInput
  └── 立即充值 Button

PayDetailPage（缴费详情）
  └── walletCard Builder
       ├── 电子钱包标题 + 余额
       ├── 抵扣金额 + Toggle 开关
       └── 状态：enableWallet（余额>0可开启）
```

### 1.3 状态管理方案
- 使用 `@ObservedV2` / `@Trace` 装饰器实现响应式状态
- 全局余额通过 `GlobalUtils`（AppStorageV2 连接）跨模块共享
- `MyWalletViewModel` 采用单例模式，确保全局状态一致性
- 跨模块事件通信：`EmitUtils.sendEmit('onConsumeWallet')` / `onEmit`

## 2. 页面组件实现

### 2.1 MyWalletPage（钱包详情页）
- **组件名称**：`MyWalletPage`
- **文件路径**：`features/business_mine/src/main/ets/pages/MyWalletPage.ets`
- **Builder 导出**：`MyWalletPageBuilder`
- **UI 结构**：
  - `NavDestination` 容器，`hideTitleBar(true)`
  - 顶部余额区域：Column + Row 布局，背景图 `bg_self_point_title`
  - 筛选区域：年月选择器 + Tab 标签，背景图 `bg_wallet_month`
  - 列表区域：`List` + `ForEach` 渲染变动明细
  - 空状态：`EmptyComp` 组件
  - 日期选择弹窗：`bindSheet` + `TextPicker` 双列滚轮
- **状态管理**：
  - `@Local vm: MyWalletViewModel = MyWalletViewModel.instance`（单例）
  - `@Computed get walletFilterList()`：根据 actionType 筛选 walletList
- **事件处理**：
  - 充值按钮 → `RouterUtil.pushPathByName(RECHARGE_WALLET_PAGE)` → 回调 `vm.chargeWallet()`
  - 确认月份 → `vm.filterWalletListByMonth()`
  - Tab 切换 → `vm.actionType = v.value`

### 2.2 RechargeWalletPage（充值页面）
- **组件名称**：`RechargeWalletPage`
- **文件路径**：`features/business_mine/src/main/ets/pages/RechargeWalletPage.ets`
- **Builder 导出**：`RechargeWalletPageBuilder`
- **UI 结构**：
  - `NavDestination` 容器
  - 序列号 `TextInput`（InputType.Number）
  - 密钥 `TextInput`
  - "立即充值" Button
- **状态管理**：`@Local vm: RechargeWalletViewModel = new RechargeWalletViewModel()`
- **事件处理**：
  - `validateInput()` 校验序列号和密钥非空
  - 校验通过后 `RouterUtil.pop(1)` 返回上一页

### 2.3 MinePage 钱包入口卡片
- **位置**：`features/business_mine/src/main/ets/pages/MinePage.ets` → `mineCard()` Builder
- **UI 结构**：
  - Row({ space: 8 }) 内包含两个 layoutWeight(1) 的 Row
  - 钱包卡片：图标 `ic_mine_wallet` + "我的钱包" + 余额 `￥{globalInfo.myWallet}`
  - 背景图：`bg_wallet`
- **事件处理**：点击 → `RouterUtil.pushPathByName(RouterMap.MY_WALLET_PAGE)`

### 2.4 PayDetailPage 钱包支付卡片
- **位置**：`features/business_home/src/main/ets/pages/PayDetailPage.ets` → `walletCard()` Builder
- **UI 结构**：
  - "电子钱包支付"标题
  - Row 内左侧：电子钱包名称 + 余额显示
  - Row 内右侧：抵扣金额 `FormatUtils.formatCurrency(useWalletMoney)` + Toggle 开关
- **状态管理**：
  - `this.vm.useWallet`：Toggle 绑定状态
  - `this.vm.enableWallet`：Toggle enabled 状态（余额 > 0）
  - `this.vm.useWalletMoney`：自动计算的抵扣金额
- **事件处理**：Toggle onChange → `this.vm.useWallet = isOn`

## 4. 数据模型

### 4.1 IWallet 接口 [核心]
```typescript
export interface IWallet {
  id?: string      // 索引（可选，自动生成时间戳）
  name: string     // 名称（如"充值"、"停车缴费"）
  time: number     // 生成时间（时间戳）
  action: number   // 充值1、支出2
  money: string    // 金额（字符串格式）
}
```

### 4.2 Wallet 类 [核心]
```typescript
@ObservedV2
export class Wallet {
  id: string = ''
  name: string = ''
  time: number = 0
  action: number = 1    // 充值1、支出2
  money: string = ''

  constructor(item: IWallet) {
    this.id = item.id ?? new Date().getTime().toString()
    this.name = item.name
    this.time = item.time
    this.action = item.action
    this.money = item.money
  }
}
```

### 4.3 WalletType 枚举 [核心]
```typescript
export enum WalletType {
  ALL = 0,   // 全部
  IN,        // 充值（值为1）
  OUT,       // 支出（值为2）
}
```

### 4.4 WalletConstants [辅助]
```typescript
export interface WalletItem {
  title: string,   // Tab 标题
  value: number,   // 对应 WalletType 值
}

export class WalletConstants {
  static readonly TAB_OPTIONS: WalletItem[] = [
    { title: '全部', value: 0 },
    { title: '充值', value: 1 },
    { title: '支出', value: 2 },
  ]
}
```

### 4.5 GlobalUtils.myWallet [核心]
```typescript
@ObservedV2
export class GlobalUtils {
  @Trace myWallet: number = 200   // 初始余额200元
}
```

## 5. 网络请求

### 5.1 获取钱包明细 [核心]
- **URL**：`getMyWalletList`
- **方法**：GET
- **请求参数**：`{ storeId: string, userId: string }`
- **响应结构**：`BaseResponse<Array<Wallet>>`
- **API 封装**：`HttpRequestApi.getMyWalletList()`
- **业务封装**：`HttpRequest.getMyWalletList()` → `Promise<Array<Wallet>>`
- **Mock 数据**：8 条记录（4笔充值：180/50/80/20，4笔停车缴费：15/15/30/10）

### 5.2 充值 [核心]
- 前端模拟，不调用后端 API
- `MyWalletViewModel.chargeWallet()` 直接修改本地状态：
  1. 构造 `IWallet { name:'充值', time:Date.now(), action:1, money:'100' }`
  2. 插入到 `allWalletList` 头部
  3. `globalInfo.myWallet += 100`
  4. 调用 `filterWalletListByMonth()` 刷新列表

### 5.3 消费 [核心]
- 前端模拟，不调用后端 API
- 事件驱动：`EmitUtils.sendEmit('onConsumeWallet', { data: { money } })`
- `IndexViewModel.onConsumeWallet()` 全局监听 → `MyWalletViewModel.consumeWallet(money)`
  1. 构造 `IWallet { name:'停车缴费', time:Date.now(), action:2, money:money.toString() }`
  2. 插入到 `allWalletList` 头部
  3. `globalInfo.myWallet -= money`
  4. 调用 `filterWalletListByMonth()` 刷新列表
