# 店铺导购 需求PRD

> 版本：1.0.0 | 平台：HarmonyOS | 更新日期：2026-04-30

## 1. 业务场景
商场用户在逛商场时，需要快速查找店铺位置、了解店铺详细信息（名称、Logo、楼层、分类、电话、简介等）。用户可能不熟悉商场布局，需要通过楼层浏览、分类筛选、关键词搜索等方式定位目标店铺，并查看店铺详情以获取联系方式和位置信息。

## 2. 业务目标
- 为用户提供便捷的商场店铺导览能力，提升用户在商场内的购物体验
- 通过楼层和分类多维度筛选，帮助用户快速定位目标店铺
- 提供店铺详情信息（位置、电话、简介），方便用户联系和到店

## 3. 功能需求

### 3.1 店铺导购列表页 [P0]
- 实现店铺导购列表页，支持从首页金刚区"店铺导购"图标导航进入
- 实现楼层轮播图展示（B1~5F），用户可点击楼层图片跳转到对应楼层的店铺详情 [P0]
- 实现关键词搜索功能，用户可在搜索栏输入店铺名称进行筛选 [P1]
- 实现分类标签横向滚动列表（全部、餐饮、服装、儿童、精品、娱乐、母婴、运动），各标签显示对应数量 [P0]
- 实现楼层侧边栏筛选（全部楼层、5F、4F、3F、2F、1F、B1） [P0]
- 实现店铺网格卡片列表，展示店铺 Logo、名称、位置信息 [P0]
- 支持分类和楼层组合筛选 [P0]
- 筛选结果为空时展示空状态提示 [P1]

### 3.2 店铺详情页 [P0]
- 实现店铺详情页，从导购列表点击店铺卡片导航进入
- 展示店铺大图（bigImage） [P0]
- 展示店铺 Logo、名称 [P0]
- 展示店铺标签（支持多个标签以逗号分隔展示） [P0]
- 展示店铺位置信息 [P0]
- 展示店铺电话，点击电话区域弹出底部拨号面板 [P0]
- 展示店铺简介 [P1]

### 3.3 首页入口 [P0]
- 实现首页金刚区"店铺导购"图标入口
- 实现首页"精选推荐"区域点击广告图片可跳转到对应店铺详情页 [P1]

## 5. 交互流程

### 5.1 店铺导购主流程 [P0]
1. 用户在首页金刚区点击"店铺导购"图标
2. 系统导航至店铺导购列表页（StoreGuidePage）
3. 页面加载时自动请求店铺数据 API
4. 用户可点击顶部楼层轮播图跳转到对应楼层店铺详情
5. 用户可通过分类标签筛选店铺
6. 用户可通过左侧楼层侧边栏筛选店铺
7. 用户可在搜索栏输入关键词搜索店铺
8. 用户点击店铺卡片，导航至店铺详情页（StoreGuideDetailPage）

### 5.2 精选推荐跳转流程 [P1]
1. 首页加载时请求店铺列表数据
2. 用户点击"精选推荐"区域的广告图片
3. 系统根据广告图片关联的店铺 ID 匹配店铺数据
4. 匹配成功则导航至对应店铺详情页

### 5.3 店铺电话拨打流程 [P0]
1. 用户在店铺详情页点击电话区域
2. 系统弹出底部拨号面板（CallTelSheet）
3. 用户可通过面板拨打店铺电话

## 6. 功能边界与限制条件

- 店铺数据通过 API 从服务端获取，使用商场 ID（storeId）作为查询参数 [P0]
- 分类标签固定为：全部、餐饮、服装、儿童、精品、娱乐、母婴、运动 [P0]
- 楼层固定为：B1、1F~5F [P0]
- 楼层轮播图使用本地预设图片资源 [P1]
- 店铺图片（Logo、大图）使用云端存储 URL 前缀拼接 [P0]
- 搜索仅支持店铺名称模糊匹配 [P1]
- 筛选无结果时展示"暂无相关店铺信息"空状态提示 [P1]
- 首页金刚区导购图标固定为 3 列布局（删除后需适配） [P0]
