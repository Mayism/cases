# 店铺导购 技术实现

> 版本：1.0.0 | 平台：HarmonyOS | 更新日期：2026-04-30

## 1. 技术架构

### 模块结构
店铺导购功能位于 `features/business_home` 模块内，属于 business_home HAR 模块的子功能。入口通过 `products/phone`（entry HAP）中的 HomePage 金刚区图标导航进入。

```
products/phone (entry HAP)
  └── HomePage.ets
       ├── 金刚区 "店铺导购" 图标 → RouterMap.STORE_GUIDE_PAGE
       └── "精选推荐" 广告图 → RouterMap.STORE_GUIDE_DETAIL_PAGE

features/business_home (HAR)
  ├── pages/StoreGuidePage.ets        (导购列表页)
  ├── pages/StoreGuideDetailPage.ets  (详情页)
  ├── viewmodel/StoreGuideViewModel.ets
  ├── viewmodel/StoreGuideDetailViewModel.ets
  └── components/GuideListComp.ets

commons/lib_common (HAR)
  ├── models/ResponseModel.ets    (StoreGuide, GuideTag)
  ├── models/RouterModel.ets      (GuideRouter)
  ├── constants/RouterMap.ets     (路由常量)
  ├── constants/Common.ets        (API URL)
  ├── httprequest/HttpRequest.ets (getStoreGuide)
  ├── httprequest/HttpRequestApi.ets
  └── mock/MockData.ets           (Mock 数据)
```

### 组件层级
```
StoreGuidePage
  ├── NavHeaderBar
  ├── Swiper (楼层图片轮播)
  └── GuideListComp
       ├── Search (搜索栏)
       ├── List (分类标签)
       ├── List (楼层侧边栏) → TitleItem 组件
       └── Grid (店铺卡片网格) / EmptyComp

StoreGuideDetailPage
  ├── NavHeaderBar
  ├── Image (大图)
  ├── Image (Logo) + Text (名称) + Flex (标签)
  ├── 位置信息
  ├── 电话信息 → bindSheet → CallTelSheetBuilder
  └── 店铺简介
```

### 状态管理方案
使用 `@ObservedV2` + `@Trace` 装饰器实现响应式状态管理。ViewModel 持有业务数据，页面组件通过 `@Local` 引用 ViewModel 实例。

## 2. 页面组件实现

### 2.1 StoreGuidePage（店铺导购列表页）
- **文件路径**: `features/business_home/src/main/ets/pages/StoreGuidePage.ets`
- **组件类型**: `@ComponentV2 struct StoreGuidePage`
- **Builder 导出**: `StoreGuidePageBuilder()`
- **状态管理**: `@Local vm: StoreGuideViewModel`
- **UI 结构**: NavDestination > NavHeaderBar + Scroll > Column > Swiper + GuideListComp
- **生命周期**: `aboutToAppear()` 调用 `this.vm.getService()` 加载数据
- **Swiper**: 水平展示楼层图片，支持循环、指示器、onChange 更新索引
- **楼层点击**: 通过 `vm.goStore(item)` 查找对应店铺并导航到详情页

### 2.2 StoreGuideDetailPage（店铺详情页）
- **文件路径**: `features/business_home/src/main/ets/pages/StoreGuideDetailPage.ets`
- **组件类型**: `@ComponentV2 struct StoreGuideDetailPage`
- **Builder 导出**: `StoreGuideDetailPageBuilder()`
- **状态管理**: `@Local vm: StoreGuideDetailViewModel`, `@Local mallModel: MallModel`
- **计算属性**: `@Computed get tagArr()` 将逗号分隔的 tag 字符串拆分为数组
- **生命周期**: `aboutToAppear()` 调用 `this.vm.getParams()` 从路由参数获取店铺数据
- **图片 URL**: 使用 `CommonUrl.CLOUD_STORAGE_URL` 前缀拼接 `bigImage` 和 `logo`
- **电话面板**: 使用 `bindSheet` 绑定 `CallTelSheetBuilder`，传入商场电话号码

### 2.3 GuideListComp（导购列表组件）
- **文件路径**: `features/business_home/src/main/ets/components/GuideListComp.ets`
- **组件类型**: `@ComponentV2 export struct GuideListComp`
- **参数**: `@Param @Require vm: StoreGuideViewModel`
- **计算属性**:
  - `filterGuides`: 根据 searchText、searchTag、searchFloor 三重过滤店铺列表
  - `tagListShow`: 为每个分类标签计算当前筛选条件下的店铺数量
- **子组件 TitleItem**: `@ComponentV2 export struct TitleItem`，楼层侧边栏项，支持选中态高亮
- **交互**: 点击分类标签更新 vm.searchTag，点击楼层项更新 vm.searchFloor，点击店铺卡片导航到详情页

### 2.4 StoreGuideViewModel
- **文件路径**: `features/business_home/src/main/ets/viewmodel/StoreGuideViewModel.ets`
- **类类型**: `@ObservedV2 export class StoreGuideViewModel`
- **响应式状态** (@Trace): storeGuideList, floorImages, searchText, searchTag, searchFloor, swiperIndex
- **静态数据**: tagList（8个分类）, floorList（7个楼层）
- **Floor 内部类**: text(位置编码), img(图片资源), floor(楼层标识)
- **方法**:
  - `getService()`: 调用 `HttpRequest.getStoreGuide()` 获取店铺列表
  - `getFloorGuideInfo(item)`: 根据楼层图片查找匹配的店铺
  - `goStore(item)`: 导航到店铺详情页，路由参数为 `GuideRouter`

### 2.5 StoreGuideDetailViewModel
- **文件路径**: `features/business_home/src/main/ets/viewmodel/StoreGuideDetailViewModel.ets`
- **类类型**: `@ObservedV2 export class StoreGuideDetailViewModel extends WindowModel`
- **响应式状态** (@Trace): storeGuide (StoreGuide), showCallSheet (boolean)
- **方法**: `getParams()` 从路由参数中提取 GuideRouter，逐字段赋值到 storeGuide

## 3. 关键API使用

- [核心] `@kit.ArkUI.AppStorageV2` — 全局状态连接，用于 MallModel（商场信息）
- [核心] `NavDestination` + Navigation 路由系统 — 页面导航，通过 `RouterUtil.pushPathByName` 实现
- [核心] `@ohos/axios` — HTTP 请求框架，通过 AxiosRequest 封装
- [辅助] `@ObservedV2` / `@Trace` / `@Computed` — 响应式状态管理
- [辅助] `@ComponentV2` / `@Param` / `@Require` / `@Event` — 组件通信

## 5. 网络请求

### [核心] getStoreGuide — 获取店铺导购列表
- **URL**: `getStoreGuide`
- **方法**: GET
- **请求参数**: `{ storeId: string }`（从 MallModel.shoppingMall.id 获取）
- **响应结构**: `BaseResponse<Array<StoreGuide>>`
  - `code: number` (0 = 成功)
  - `msg: string`
  - `data: Array<StoreGuide>`
- **调用链**: `StoreGuideViewModel.getService()` → `HttpRequest.getStoreGuide()` → `HttpRequestApi.getStoreGuide()` → `AxiosRequest.get()`
- **Mock 数据**: 返回 24 条店铺记录，覆盖 B1~5F 各楼层，每楼层 3-9 条

## 6. 路由配置

### [核心] route_map_home.json 路由条目
```json
{
  "name": "StoreGuidePage",
  "pageSourceFile": "src/main/ets/pages/StoreGuidePage.ets",
  "buildFunction": "StoreGuidePageBuilder"
},
{
  "name": "StoreGuideDetailPage",
  "pageSourceFile": "src/main/ets/pages/StoreGuideDetailPage.ets",
  "buildFunction": "StoreGuideDetailPageBuilder"
}
```

### [核心] RouterMap 枚举
```typescript
// commons/lib_common/src/main/ets/constants/RouterMap.ets
STORE_GUIDE_PAGE = 'StoreGuidePage',
STORE_GUIDE_DETAIL_PAGE = 'StoreGuideDetailPage',
```

### [核心] 路由参数
- 导航到 StoreGuideDetailPage 时传递 `GuideRouter` 类型参数: `{ guide: StoreGuide }`
- 使用 `RouterUtil.pushPathByName(RouterMap.STORE_GUIDE_DETAIL_PAGE, { guide: storeInfo } as GuideRouter)`
- 详情页通过 `RouterUtil.getParamByName(RouterMap.STORE_GUIDE_DETAIL_PAGE)` 获取参数
